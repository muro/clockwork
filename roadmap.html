Roadmap

Uhrzeit Lernen

Future work for turning the clock-reading game into a language-aware, installable practice app that still keeps its zero-build simplicity.

Current Principle
One app, multiple learning languages.

Keep clock mechanics shared; let phrase generation, distractors, and mastery state vary by language.

Workstreams

Language Support Complete

  • Add an in-app language selector for German and English.
  • Keep learning progress language-aware so word-based mastery does not carry across languages accidentally.
  • Move clock-word generation behind a language-aware phrase layer.
  • Add language-specific distractor rules, for example English forms such as "twenty to twelve" for 11:40.
  • Extend the browser tests to cover word options, phrase generation, progress storage, and the UI selector contract per language.

Offline PWA Support

  • Make the installed PWA work reliably without a network connection.
  • Vendor runtime dependencies that are currently loaded from CDNs, while keeping the zero-build workflow.
  • Add a service worker that caches the app shell, engine, images, icons, manifest, fonts, and local scripts.
  • Define a simple cache versioning/update strategy so installed copies do not get stuck on stale assets.

Suggested Sequence

Phrase Layer

Extract word rendering from direct German-only helpers into a small language-aware API.

Language State

Persist selected language and scope current level plus mastery by language.

English Curriculum QA

Craft English phrase examples and distractors, then expand tests around those contracts.

Offline Shell

Vendor runtime assets and add service-worker caching once the app shell is self-contained.

State Model Sketch

The selected language should feed both navigation and mastery. Clock math can remain shared, while word rendering and distractor policy branch by language.
